# CLAUDE.md / ルール最適化

セッションログからユーザーの一貫した行動パターンを検出し、CLAUDE.md やルールファイルの改善を提案する。

## 動的コンテキスト

```
既存ルール: !`ls ~/.config/claude/rules/*.md 2>/dev/null`
CLAUDE.md 行数: !`wc -l < ~/.claude/CLAUDE.md 2>/dev/null || echo "0"`
最終更新: !`stat -f "%Sm" ~/.claude/CLAUDE.md 2>/dev/null || echo "不明"`
```

---

## Phase 1: データ抽出

### 1-1. セッションログからの行動パターン抽出

ヘルパースクリプトを実行する:

```bash
~/.config/claude/skills/rules-optimizer/scripts/extract-habits.sh $ARGUMENTS
```

`$ARGUMENTS` が空の場合はデフォルトの 20 セッションを分析する。

### 1-2. 現在のルール読み込み

以下のファイルを読み込む:
- `~/.claude/CLAUDE.md`（グローバル指示）
- `~/.config/claude/rules/*.md`（全ルールファイル）

---

## Phase 2: ギャップ分析

抽出した行動パターンと既存ルールを比較し、以下を特定する。

### 2-1. 未文書化パターン（追加候補）

セッションログで一貫して見られるが、ルールに記載されていないパターン:
- 繰り返し出現するユーザー指示（「日本語で」「テスト追加して」等）
- 一貫したツール使用パターン（常に特定ツールを優先的に使用等）
- プロジェクト横断の共通設定や好み

### 2-2. 形骸化ルール（削除・更新候補）

ルールに記載されているが、実際のセッションで守られていない or 使われていないもの:
- セッションログの行動とルールが矛盾するケース
- 参照されたことがないルール

### 2-3. 改善可能なルール

既存ルールをより具体的・効果的にできるもの:
- 抽象的すぎるルールに具体例を追加
- 範囲が広すぎるルールの分割

---

## Phase 3: 提案の作成

### 提案フォーマット

各提案を以下の形式で整理する:

```markdown
## 最適化提案

### 提案 1: {タイトル}
- **種別**: 追加 / 更新 / 削除
- **対象ファイル**: {ファイルパス}
- **根拠**: {セッションログでの観察事項}
- **変更内容**:
  ```
  {追加・変更するルールテキスト}
  ```
- **影響**: {この変更で何が改善されるか}
```

### 提案の優先度付け

以下の基準で優先度を付ける:
1. **高**: セッションで毎回繰り返される指示 → ルール化で工数削減大
2. **中**: 複数セッションで見られるパターン → ルール化で一貫性向上
3. **低**: 稀だが重要なパターン → ルール化で事故防止

---

## Phase 4: ユーザーへの提示と承認

### 4-1. 分析サマリーの提示

以下を整理して提示する:

```markdown
## ルール最適化分析結果

### 現在のルール構成
- CLAUDE.md: {行数} 行
- ルールファイル: {ファイル数} 個

### 検出した改善点: {N} 件

| # | 種別 | 対象 | 提案 | 優先度 |
|---|------|-----|------|--------|
| 1 | 追加 | {ファイル} | {概要} | 高 |

### 詳細
{各提案の詳細}
```

### 4-2. 承認フロー

ユーザーに以下を確認する:
1. どの提案を適用するか（個別に選択可能）
2. 提案内容の調整が必要か
3. ファイル編集の承認

**重要: ユーザーの承認なしにファイルを編集しない。**

---

## Phase 5: ルール更新

承認された提案について:
1. 対象ファイルを Edit ツールで更新
2. 新規ルールファイルが必要な場合は作成
3. 変更前後の diff を報告

---

## 制約事項（厳守）

- **読み取り専用フェーズ**: Phase 1〜4 ではファイルの作成・編集を行わない
- **承認ゲート必須**: Phase 4 のユーザー承認をスキップしない
- **既存ルールの保護**: 承認なしに既存ルールを削除・上書きしない
- **CLAUDE.md の行数制限**: CLAUDE.md は 200 行以内を維持する（超過する場合は別ファイルへ分離を提案）
- **機密情報の除外**: セッションログ内の機密情報を提案に含めない
- **Write 禁止**: 既存ファイルの全上書きではなく、Edit で差分更新のみ行う
