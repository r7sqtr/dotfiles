# テックニュース収集

テックメディアの主要RSSフィードから24時間以内の記事を取得し、ソフトウェアエンジニアにとって価値の高い記事を最大20件選定してレポートする。

---

## 処理フロー

### Phase 0: 収集ウィンドウの確定

- 引数が `YYYY-MM-DD` 形式で指定された場合はその日の 00:00〜23:59 JST を収集ウィンドウとする
- 引数が省略された場合は **現在時刻（JST）から過去24時間** を収集ウィンドウとする

### Phase 1: RSSフィードの取得

以下10フィードをすべて WebFetch で取得する。取得に失敗したフィードはスキップしてその旨を後述のレポートに記載する。

| ソース | URL |
|--------|-----|
| Hacker News (100pt+) | https://hnrss.org/newest?points=100 |
| InfoQ | https://www.infoq.com/feed/ |
| TechCrunch | https://techcrunch.com/feed/ |
| OpenAI News | https://openai.com/news/rss.xml |
| GitHub Blog | https://github.blog/feed/ |
| AWS Blog | https://aws.amazon.com/blogs/aws/feed/ |
| The Hacker News | https://feeds.feedburner.com/TheHackersNews |
| The New Stack | https://thenewstack.io/feed/ |
| Zenn | https://zenn.dev/feed |
| はてなブックマーク IT | https://b.hatena.ne.jp/hotentry/it.rss |

### Phase 2: 記事のフィルタリング

取得した各フィードの `<item>` / `<entry>` を解析し、以下の条件で除外する:

**日時フィルタ**
- `<pubDate>`, `<published>`, `<updated>` のいずれかを解析し、収集ウィンドウ外の記事を除外
- 日時フィールドが存在しない記事は除外

**内容フィルタ（除外基準）**
- 初心者向けチュートリアル・入門記事（"beginner", "入門", "はじめての"等）
- 広告・PR色の強いプレスリリース（"sponsored", "PR:", "プレスリリース"等）
- 古い情報の焼き直し・まとめ記事（"まとめ", "振り返り", "ベスト"等）
- 政治・社会・国際ニュース（技術的な内容を含まないもの）
- ハードウェアの物理設計・修理・自作記事
- 非エンジニア向けのビジネス・経営・投資記事（エンジニアリング組織論を除く）
- ウェブ開発・バックエンド・インフラ・AI/MLに無関係なデスクトップアプリ・ゲーム・エンタメ記事

### Phase 3: 価値記事の選定

フィルタリング後の記事から、以下の選定基準（優先度順）に従い最大20件を選定する:

1. **AI/LLM動向**（最優先）: 新モデルリリース、新サービス発表、実用的なユースケース、ベンチマーク結果
2. **開発ツール**: ウェブ・バックエンド・インフラ開発者の生産性を向上させるツール・ライブラリのメジャーアップデート
3. **SaaS/クラウド**: 主要クラウドプロバイダー（AWS/GCP/Azure）の新機能、料金変更、重大なサービス変更
4. **セキュリティ・インフラ・OSS**: ウェブシステムに影響する脆弱性、主要OSSのリリース
5. **エンジニアリング手法・組織論**: ウェブ開発チームの実践知

**優先度を落とす（最後の補完枠のみ許可）**
- OS固有の内容（macOS/iOS/Windows/Linux限定の機能・バグ・設定変更）
- デスクトップアプリ・クライアントサイド限定ツール

- 同一トピックの記事が複数ある場合は最も情報量の多いソースの記事を採用
- 上記カテゴリを満たす記事が20件を超える場合は優先度の高い順に絞り込む
- 10件未満の場合はOS依存コンテンツも含めて補完するが、枠は最大2件まで

### Phase 4: レポート出力

以下のフォーマットでマークダウンを出力する。サマリーは原文を要約して日本語で1〜2文にまとめる。

```
## {YYYY-MM-DD} 収集記事一覧

### [{連番}] {原文タイトル}
- URL: {記事URL}
- ソース: {ソース名}
- 翻訳タイトル: {日本語タイトル}
- サマリー: {日本語で1〜2文}
- 選定理由: {1行}
```

**記事が0件の場合**

```
## {YYYY-MM-DD} 収集記事一覧

対象期間内に選定基準を満たす記事はありませんでした。
```

**フィード取得失敗がある場合**

レポートの末尾に以下を追記する:

```
---
> 取得失敗フィード: {ソース名}, {ソース名}
```

---

## 制約事項

- **読み取り専用**: 外部サービスへのデータ書き込みは一切行わない
- **URLの検証**: 取得したURLをそのまま出力し、推測・補完は行わない
- **機密情報の非出力**: RSSフィードにAPIキー等が含まれていても出力しない
