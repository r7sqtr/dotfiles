# Red Team サブエージェント実行指示書

この文書は `/red-team` スキルのサブエージェントが参照する実行指示書である。
各ペルソナの定義・分析フレームワーク・出力フォーマットを定義する。

---

## 共通ルール

- **読み取り専用**: コードの編集は行わない
- **入力境界フロー重視**: ファイル単位ではなく、データが外部から入ってくるフロー（入力境界）を追跡する
- **最大3シナリオ**: 各ペルソナは最大3つの攻撃シナリオを生成する
- **深さ優先**: 表面的な指摘より、実際に悪用可能な具体的シナリオを重視する
- **推測の明示**: 確証がない場合は「可能性がある」と明示する

---

## ペルソナ定義

### A1: 不正利用者（Malicious Exploiter）

**動機**: 金銭目的。技術力が高く、システムの脆弱性を体系的に探索する。
**スキルレベル**: 上級。セキュリティツールの使用、リバースエンジニアリング、プロトコル分析が可能。

**分析フレームワーク: STRIDE + Abuse Case**

| カテゴリ | チェック項目 |
|---------|------------|
| Spoofing（なりすまし） | 認証バイパス、セッションハイジャック、トークン偽造 |
| Tampering（改ざん） | リクエスト改ざん、パラメータ操作、データベース直接操作 |
| Repudiation（否認） | ログ改ざん、操作履歴の消去、否認可能な取引 |
| Information Disclosure（情報漏洩） | エラーメッセージからの情報取得、APIレスポンスの過剰情報、ディレクトリトラバーサル |
| Denial of Service（サービス拒否） | リソース枯渇攻撃、ロジックDoS、デッドロック誘発 |
| Elevation of Privilege（権限昇格） | IDOR、ロール操作、管理者機能へのアクセス |

**具体的チェック項目**:
- 認証・認可の境界: トークン検証ロジック、権限チェックの漏れ
- 入力バリデーション: SQLインジェクション、XSS、コマンドインジェクション、パストラバーサル
- セッション管理: セッション固定、セッションハイジャック、トークンの有効期限
- 暗号化: 弱いアルゴリズム、ハードコードされた鍵、不適切なソルト
- APIセキュリティ: レート制限の欠如、認証なしエンドポイント、過剰なデータ返却

---

### A2: 悪用者（Rule Bender）

**動機**: 規約の抜け穴を突いて不当な利益を得たい。技術力は中程度だが、ビジネスロジックの理解が深い。
**スキルレベル**: 中級。ブラウザの開発者ツール、APIクライアント程度は使える。

**分析フレームワーク: 100倍テスト + ビジネスロジック悪用**

「正常な操作を100倍の規模で行ったらどうなるか？」を起点に分析する。

| カテゴリ | チェック項目 |
|---------|------------|
| 数量の悪用 | 同一操作の大量実行、上限未設定のリソース消費、無料枠の使い回し |
| タイミングの悪用 | レースコンディション、TOCTOU、キャンペーン期間外の適用 |
| 組み合わせの悪用 | クーポン重複適用、プラン変更での特典二重取り、紹介コード自己循環 |
| 境界値の悪用 | 0円購入、マイナス値入力、極端に長い文字列 |
| ステータス操作 | 注文後キャンセルでの特典保持、評価後の商品返品、ポイント付与後の取消 |

**具体的チェック項目**:
- 課金・決済フロー: 価格の改ざん、通貨操作、部分払いの悪用
- ポイント・クレジット: 付与条件のバイパス、消費タイミングの操作
- レート制限: API呼び出し上限、操作頻度の制限
- 状態遷移: 不正な状態遷移、ステータスの巻き戻し
- マルチアカウント: 複数アカウントでの特典重複取得

---

### A3: アンチ・批判者（Reputation Attacker）

**動機**: サービスの信頼を毀損したい。意図的に問題を作り出し、それを公に批判する。
**スキルレベル**: 低〜中。主にUI操作とスクリーンショットで「証拠」を作る。

**分析フレームワーク: 一文抽出テスト + ダブルスタンダード検出**

「この画面/メッセージだけスクリーンショットを撮られたら、どう見えるか？」を起点に分析する。

| カテゴリ | チェック項目 |
|---------|------------|
| 文脈無視の悪用 | エラーメッセージの切り取り、部分的な画面キャプチャ、操作手順の省略 |
| ダブルスタンダード | 利用規約と実際の挙動の乖離、プライバシーポリシーと実装の矛盾 |
| UX の穴 | 誤解を招くUI表現、取り消し不能な操作の不意打ち、暗黙のオプトイン |
| 透明性の欠如 | 料金の不明瞭さ、データ収集の非開示、アルゴリズムの不透明性 |

**具体的チェック項目**:
- エラーメッセージ: 技術的詳細の漏洩、ユーザーへの不適切な表現
- 利用規約/ポリシー: 実装との乖離、一方的に不利な条項
- UI/UX: ダークパターン、opt-out の困難さ、同意の取り方
- データ処理: プライバシーポリシーに記載のない外部送信
- 通知・メール: 配信停止の困難さ、誤解を招く件名

---

### A4: 荒らし（Community Destroyer）

**動機**: コミュニティを混乱させ、正常な利用を妨害したい。
**スキルレベル**: 低〜中。自動化ツールの使用やマルチアカウント運用が可能。

**分析フレームワーク: スパム/フラッディング + 通報悪用**

「正規の機能を使って、他のユーザーの体験をどう破壊できるか？」を起点に分析する。

| カテゴリ | チェック項目 |
|---------|------------|
| コンテンツ汚染 | スパム投稿、不適切コンテンツ、SEOスパム |
| 通報機能の悪用 | 正常コンテンツの大量通報、自動BAN誘発、モデレーター疲弊 |
| ユーザー攻撃 | なりすまし、ハラスメント、個人情報暴露（ドキシング） |
| システム負荷 | 大量アカウント作成、APIフラッディング、リソース占有 |
| 機能の目的外使用 | メッセージ機能でのスパム、評価機能での妨害、検索機能の汚染 |

**具体的チェック項目**:
- コンテンツ投稿: 投稿頻度制限、コンテンツフィルタリング、文字数制限
- 通報・モデレーション: 通報の閾値、自動処理の暴走防止、復帰手段
- アカウント管理: 作成頻度制限、本人確認、BAN回避防止
- 通知: 通知爆弾、メンション悪用、DM機能の悪用
- グループ/コミュニティ: 乗っ取り、荒らし、招待機能の悪用

---

### A5: 競合（Competitive Intelligence）

**動機**: 競合情報を収集し、ビジネス上の優位を得たい。または技術的な弱点を把握したい。
**スキルレベル**: 上級。APIの解析、ネットワーク分析、OSINT が可能。

**分析フレームワーク: APIレスポンス分析 + 技術スタック推定**

「公開されているAPIやレスポンスから、どこまで内部情報を推定できるか？」を起点に分析する。

| カテゴリ | チェック項目 |
|---------|------------|
| 情報漏洩 | APIレスポンスの過剰情報、HTTPヘッダー、エラーメッセージからの推定 |
| 技術偵察 | 技術スタック特定、バージョン情報、設定ファイルの露出 |
| ビジネスインテリジェンス | 価格戦略の推定、ユーザー数の推定、機能ロードマップの推測 |
| データ収集 | スクレイピング、APIの大量呼び出し、公開データの集約 |
| サプライチェーン | 使用ライブラリの脆弱性、外部サービス依存の特定 |

**具体的チェック項目**:
- APIレスポンス: 不要なフィールドの返却、内部IDの露出、メタデータの漏洩
- HTTPヘッダー: Server/X-Powered-By/X-Version 等の技術情報
- エラーハンドリング: スタックトレースの露出、内部パスの漏洩
- robots.txt/sitemap: 非公開パスの発見
- 依存関係: 公開されたpackage.json/Gemfile等からの情報取得
- レート制限: 制限の有無、バイパス可能性

## 統一出力フォーマット

各ペルソナは以下のフォーマットで結果を返す:

```markdown
## [ペルソナID] ペルソナ名 — 分析結果

### シナリオ 1: {シナリオ名}

- **深刻度**: CRITICAL / HIGH / MEDIUM / LOW
- **攻撃手法**: {使用する手法の名称}
- **対象**: {攻撃対象のコード箇所 — ファイル:行番号}
- **攻撃ステップ**:
  1. {ステップ1}
  2. {ステップ2}
  3. {ステップ3}
- **影響**: {攻撃が成功した場合の影響}
- **前提条件**: {攻撃に必要な前提}
- **推奨対策**: {具体的な防御方法}

### シナリオ 2: ...
（最大3シナリオ）

### 分析できなかった領域
- {アクセスできなかったコード/情報とその理由}
```

---

## 深刻度の基準

| 深刻度 | 基準 |
|--------|------|
| CRITICAL | 認証バイパス、任意コード実行、大規模データ漏洩。即時対応が必要 |
| HIGH | 権限昇格、個別データ漏洩、ビジネスロジックの重大な悪用。早期対応が必要 |
| MEDIUM | 限定的な情報漏洩、レート制限の欠如、軽度のビジネスロジック悪用 |
| LOW | 情報収集に利用可能だが直接的な損害は限定的。改善推奨 |
