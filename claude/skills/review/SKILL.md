# コードレビュー

指定されたコード（$ARGUMENTS）を批判的にレビューしてください。

## レビュー観点

### 1. 可読性
- 変数名・関数名はわかりやすいか
- コードの意図が伝わるか
- 適切なコメントがあるか

### 2. バグの可能性
- エッジケースの考慮漏れ
- null/undefined のチェック漏れ
- 型の不整合
- ロジックの誤り

### 3. パフォーマンス
- 明らかに非効率な処理がないか
- 不要な再計算・再レンダリングがないか
- N+1問題の可能性

### 4. セキュリティ
- 入力値の検証は適切か
- 機密情報の扱いは問題ないか
- インジェクション攻撃への対策

## 制約事項（厳守）

- **ファイルの編集禁止**: 指摘のみを行い、コードを書き換えない
- **Git操作禁止**: commit, push, reset 等の状態変更コマンドは実行しない
- **指摘がない場合**: 「レビューの結果、特に指摘事項はありません」と明確に伝える

## 出力フォーマット

各指摘は以下の形式で報告：

```
### [カテゴリ] ファイル名:行番号

**問題**: 問題の説明
**理由**: なぜ問題なのか
**推奨**: どう改善すべきか
```

指摘がない場合は、その旨を簡潔に伝えてください。
